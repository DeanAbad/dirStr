Info

  • Pressing the Convert button converts the given string into directory/path and vice versa.
  • Pressing the Clear erases all fields in a converter section e.g. String to Directory.
  • Format Allows
    C:/The Cool User/Awesome Games Yes
    C:/The Cool User/Awesome Games/ No
    \C:/The Cool User/Awesome Games Yes
    \C:/The Cool User/Awesome Games/ No
    ./The Cool User/Awesome Games Yes
    • Note: Examples are applicable to directory as well: ./test here/now is also feasible with ./test%20here/now.

Update

  • Since yesterday in the evening, the Internet connection came back, and so I worked with the escape() and unescape() functions, as Intellisense tells that they are already depreciated (although not fully); I changed them into encodeURI() and decodeURI() respectively.
  • Thanks to the Internet, I was able to get some good stuff at Stack Overflow and MDN Web Docs with regex; now the converter checks for regex first.

Why make it?

  • There are times when I need the escaped string of directory to paste on certain apps, so this tool is good for that. Although, certainly, there are already countless apps out there which are superior than mine, but I am reviewing JavaScript and wanted to build it, so I did.
  • To test and apply what I have learned with freeCodeCamp's Responsive Web Design course, and see how far I have recalled or at least understood legitimately.
